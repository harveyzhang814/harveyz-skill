# Research Loop 设计原则

研究循环（research loop，多轮开放式探索任务）的方法论原则。从四轮实践里反复出现、和具体主题无关的设计决策中抽出，共八类。

## 一、范围与收录：宽进严记

- 收录标准写成"哪怕牵强也纳入"，而不是"确保质量"——精确度是收尾阶段的事，探索阶段的敌人是过早收敛。
- 但"宽进"不等于"不做判断"：凡是被排除、被搁置、被认为关联不足的，都要求**留下判断本身**，不能沉默跳过。"未决定的排除"必须变成"决定过的边界"。

## 二、起点：给一个锚点，不给一条路径

- 完全空白的探索空间会让 agent 不知从何下手，所以每轮都给一个明确的**起点**（交汇点、候选节点、具体缺口），但从第二步起就不再规定必须按什么顺序、回到哪里。
- 起点可以是多个并存的候选，不强求收敛成唯一答案（允许"部分是A部分是B"这种不干净的结论）。

## 三、交付物：按功能拆分，禁止互相塌缩

- 不同性质的产出必须物理拆开，且显式声明"这份不是那份的摘要"：
  - **内容文档**（讲透单个对象本身）
  - **关系文档**（讲清楚对象之间及在脉络中的位置）
  - **过程日志**（讲清楚探索路径本身怎么走、哪里回头了）
- 三者最容易互相退化成对方的复述，所以每一份都要单独写一句"这份不得退化为XX的复述"。

## 四、过程记录：实时写，允许糙

- 日志必须在探索过程中**实时追加**，不能事后补写——事后补写的日志会变成精修叙事，失去记录真实决策路径的价值。
- 显式允许"简短、口语化"，这是为了防止 agent 把日志也写成正式文档，从而和"关系文档"重复。
- 记录内容不止"做了什么"，更要记录**"为什么放弃了什么"**——这是最容易被省略、但价值最高的部分。

## 五、延续性：新一轮先读旧产出，禁止重复造轮子

- 每轮开始前强制要求"先通读已有产出"，并给出具体应对方式：发现重合优先在已有文件里补充，而不是新建重复内容。
- 跨轮的文档体系不重启，只在同一文件里标注分界（如"第三轮"），保持单一事实来源。

## 六、执行约束：双触发 + 优雅降级，而非硬停止

- 时间和资源用两个独立指标限制（例如"总时长"和"额度消耗比例"），明确"以先到者为准"。
- 触发后不是立刻掐断，而是规定一套**收尾动作序列**：完成手头的、更新索引、日志里写清楚为什么在这停、列出没做完的清单。降级路径要写清楚，不能让 agent 到点了自己猜该怎么收。

## 七、任务优先级：明确"先砍哪个"

- 当一轮包含多个任务且时间可能不够时，显式排出优先级，并指定**第一个可舍弃项**。
- 被舍弃的任务不能默默消失，必须出现在"未展开清单"里，位置还要靠前——这和第一条"宽进严记"是同一个精神的延伸：可以不做，但不能不留痕。

## 八、检索与结论要分离

- 对"这个方向可能信息量低"这类直觉判断，不允许直接跳过，必须显式要求"先检索，后下结论"，并且检索过程本身要进研究地图。
- 目的是把"我觉得没什么"和"我查过确实没什么"区分开——前者是偷懒，后者是结论。

## 可复用模板骨架

```
目标：[领域] 中 [某个锚点/起点]，向 [N个方向] 展开，收录标准从宽：
只要存在哪怕松散/间接的关联就纳入，不强求关联被严格论证。

起点与展开：第一批节点从 [锚点] 建立，覆盖 [方向a/b/c...]；
第二批起路径完全开放，沿任意维度自由延伸，直到信息增量递减。

延续性：开始前先通读已有产出于 [路径]，避免重复；发现重合
优先补充已有文件而非新建。

交付物：
- [内容文档]：单个对象讲透，固定结构 [四段式/其他]
- [关系文档]：只讲关系与脉络，不得复述内容文档
- [过程日志]：实时追加，记录路径决策与放弃理由，允许糙

模糊地带：任何"排除/搁置/信息量低"的判断，必须先做最小限度
的核实，再记录为显式结论，不得默认跳过。

执行限制：[时间上限] 与 [资源消耗比例] 双触发，以先到为准；
触发后收尾动作为 [完成在做的/更新索引/日志写明原因/列未展开清单]。
多任务时明确优先级与第一个可舍弃项。
```
