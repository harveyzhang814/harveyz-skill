# Bain & Company 官方品牌标准

> 本文件记录 Bain & Company 官方品牌视觉语言，基于 bain.com 公开发布的视觉资产与 Bain 内部报告样式整理。
> 与 `custom-style.md`（Harvey 自定义设计系统）及 `rb-style.md`（Roland Berger 品牌标准）区分开：本文件追求还原原汁原味的 Bain 品牌风格。

---

## 1. 品牌概述

Bain & Company 官方品牌的核心特征：
- **红色作为绝对主角**：Bain Red `#CC0000` 是品牌第一色，贯穿标题、CTA、图表主色
- **黑白为底、红色点睛**：背景干净白色，正文近黑，红色出现位置极克制但极有力
- **几何无衬线主导**：字体简洁有力，追求工业感与精准感，拒绝装饰性衬线
- **数据驱动视觉**：图表密度高，色彩配比注重对比度而非美观，服务于信息传递
- **极简装饰、最大信息密度**：无渐变、无阴影、无圆角装饰；框架感靠线条与字重建立

## 2. 色彩体系

### 2.1 主色板

| 角色 | 色值 | 用途 |
|------|------|------|
| Bain Red | `#CC0000` | 第一品牌色：H1/H2 标题、CTA 按钮、图表主色、重要装饰线 |
| Deep Red | `#990000` | 深化版红色，用于按压态、深色区块标题、红色层级第二层 |
| Near Black | `#1A1A1A` | 主文字、一级标题（非红色场景） |
| Body Charcoal | `#2D2D2D` | 正文段落文字 |
| Secondary Gray | `#666666` | H4 辅助标题、图表次级标签 |

### 2.2 功能色

| 角色 | 色值 | 用途 |
|------|------|------|
| Light Background | `#F5F5F5` | 卡片背景、交替行、代码块底色 |
| Border Light | `#DDDDDD` | 分隔线、细边框 |
| Border Medium | `#BBBBBB` | 表格行间线 |
| White | `#FFFFFF` | 页面底色、深色区块内文字 |
| Dark Block | `#1A1A1A` | 封面、深色 Hero 区块背景 |

### 2.3 图表 / 数据可视化配色

Bain 图表配色以**红—灰双轴**为核心，红色始终为正向/主指标，灰色为对比/次级：

| 层级 | 色值 | 典型用途 |
|------|------|---------|
| 数据色 1（主） | `#CC0000` | 主指标、Bain 客户数据 |
| 数据色 2（次） | `#666666` | 对标企业数据、次级趋势 |
| 数据色 3 | `#BBBBBB` | 背景基准线、行业平均 |
| 数据色 4 | `#1A1A1A` | 总计/合计行、高强调线 |
| 数据色 5（辅） | `#E87722` | 橙色用于第三维度或机会区域 |
| 数据色 6（辅） | `#003366` | 深蓝用于外部对比方或竞争对手 |

**规律**：红色永远代表 Bain 侧或主推结论；灰色系承载所有"其他"；橙/蓝仅用于第三维度，不抢占红色地位。

### 2.4 语义辅助色

| 语义 | 色值 | 场景 |
|------|------|------|
| 正向 / 增长 | `#CC0000`（品牌红） | 业绩改善、推荐路径 |
| 警示 / 风险 | `#E87722`（橙） | 需要关注的风险区域 |
| 中性 / 对比 | `#666666`（中灰） | 基准对比、现状描述 |
| 负向 / 衰退 | `#1A1A1A`（近黑） | 强调问题或下降趋势 |

## 3. 字体体系

### 3.1 官方字体

| 场景 | 字体 | 说明 |
|------|------|------|
| 标题 / 品牌文字 | `Gotham` | 几何无衬线，是 Bain 最具辨识度的字体 |
| 正文 | `Gotham Book` / `Gotham Light` | 较细字重用于正文，提升可读性 |
| 数字 / 数据标签 | `Gotham` | 等宽数字特性，对齐图表数值 |
| 中文标题 | `PingFang SC` | macOS/iOS 原生中文，风格接近几何无衬线 |
| 中文正文 | `PingFang SC` | 轻字重，与 Gotham 风格协调 |

### 3.2 降级字体栈（无 Gotham 授权时使用）

**英文：** `"Gotham", "Gill Sans", "Trebuchet MS", "Helvetica Neue", "Arial", sans-serif`

**中文：** `"PingFang SC", "STHeiti", "Microsoft YaHei", sans-serif`

**混排：** `"Gotham", "PingFang SC", "Helvetica Neue", "Arial", "STHeiti", "Microsoft YaHei", sans-serif`

### 3.3 字号体系

| 元素 | 字号（近似值） | 字重 | 颜色 |
|------|-------------|------|------|
| Hero 封面标题 | 40–56pt | 700 (Bold) | `#CC0000` 或 `#FFFFFF`（深底） |
| H1 | 24–32pt | 700 | `#CC0000` |
| H2 | 16–20pt | 600 | `#CC0000` 或 `#1A1A1A` |
| H3 | 13–16pt | 600 | `#1A1A1A` |
| H4 | 11–12pt | 700 | `#666666`，UPPERCASE |
| 正文 | 10–12pt | 400 | `#2D2D2D` |
| 表格 | 9–11pt | 400（表头 700） | `#1A1A1A` |
| 图表标签 | 8–10pt | 400 | `#666666` |
| 小字 / 注释 | 8–9pt | 400 | `#888888` |

## 4. 版面与间距

| 参数 | 值 | 说明 |
|------|------|------|
| 页面最大宽度 | 1280–1440px | 报告 / 网站正文区 |
| 正文区宽度 | 680–860px | 单栏阅读适宜宽度 |
| 页边距（上/下） | 2.0–2.54cm | PDF @page / DOCX section |
| 页边距（左/右） | 2.5–3.0cm | 左侧略宽，预留装订线 |
| 段落 space_after | 6–8pt | 段落间距 |
| H1 上方间距 | 0–20pt | 首标题可贴近页眉 |
| H2 上方间距 | 20–28pt | 章节间主要留白 |
| H3 上方间距 | 12–16pt | 小节间留白 |

| 两端对齐 | 优先 | 专业报告强制两端对齐 |

## 5. 组件风格

### 5.1 标题装饰

| 级别 | 装饰方式 |
|------|---------|
| H1 | Bain Red `#CC0000` 下划线，2.5–3px；或直接红色文字无线 |
| H2 | 红色左侧色条 `#CC0000`，3px；或红色文字无装饰线 |
| H3 | 无装饰，字重 + 间距区分；偶用浅灰 `#DDDDDD` 下划线 |
| H4 | 无装饰，全大写 + letter-spacing 0.08em |

### 5.2 按钮

| 类型 | 样式 |
|------|------|
| 主要 CTA | 背景 `#CC0000` 红色，白色文字，无圆角（直角） |
| 次要 CTA | 背景 `#1A1A1A` 近黑，白色文字 |
| 边框型 CTA | `border: 2px solid #CC0000`，红色文字，透明底 |

### 5.3 表格（Bain 水平线风格）

- **无竖线**，仅水平线建立结构
- 表头上方：粗红线 `#CC0000`，2–2.5px（区别于 McKinsey 黑色）
- 表头下方：细黑线 `#1A1A1A`，1px
- 数据行之间：细灰分隔线 `#DDDDDD`，0.5px（可选，密集表格使用）
- 末行下方：细黑线 `#1A1A1A`，1px
- 表头文字：加粗，无背景色，`#1A1A1A`
- 数据行：无背景色，无斑马纹；纯白底

### 5.4 卡片 / 内容区块

- 白色背景为主：`#FFFFFF`
- 重要结论 / 洞察块：左侧红色装饰线 `#CC0000`，3.5px + 浅灰背景 `#F5F5F5`
- 数字 / KPI 卡片：大字号数字用红色 `#CC0000`，配灰色描述文字
- 深色封面区块：背景 `#1A1A1A`，白色文字，红色副标题

### 5.5 引用块

- 左侧红色装饰线 `#CC0000`，4px
- 背景 `#FFF5F5`（极浅红），或纯白
- 斜体，颜色 `#2D2D2D`，字号与正文相同或略大

### 5.6 代码块

- 背景色 `#F5F5F5`，左侧装饰线 `#CC0000`，3px
- 圆角 0（直角，与品牌几何感一致）
- 字体：`Courier New` / `JetBrains Mono`，9–10pt

### 5.7 分隔线

- 颜色 `#DDDDDD`，1px；重要章节分隔用 `#CC0000`，1.5px

## 6. 格式推导指南

读取本文件后，按以下规则生成目标格式配置（在无法使用专有字体的环境下）：

### → 文档 / 报告（DOCX / PDF）

- 主色方案：**白底近黑文字，Bain Red `#CC0000` 作标题与装饰主色**
- H1 / H2 颜色：`#CC0000`，H1 带 `#CC0000` 下划线（2.5px）
- H3 左侧色条：`#CC0000`，3px
- H4：无色条，`#666666` 全大写
- 表格表头上方：`#CC0000` 红线，2px（Bain 最显著特征）
- 表格表头下方：`#1A1A1A` 黑线，1px
- 引用块左侧装饰线：`#CC0000`，背景 `#FFF5F5`
- 代码块左侧装饰线：`#CC0000`，背景 `#F5F5F5`
- 分隔线：`#DDDDDD`
- 字体降级栈：见 3.2 节

### → Mermaid 图表

- **subgraph 颜色方案**：浅灰底 `#F5F5F5` 或深黑 `#1A1A1A`，白色文字
- **主要节点**：Bain Red `#CC0000`，白色文字 `color:#fff`
- **次要节点**：深灰 `#444444`，白色文字 `color:#fff`
- **背景/参考节点**：浅灰 `#BBBBBB`，黑色文字 `color:#000`
- **正向/机会节点**：橙色 `#E87722`，白色文字 `color:#fff`
- **对比/外部节点**：深蓝 `#003366`，白色文字 `color:#fff`
- **流程图边**：主路径用红色 `#CC0000`，次路径用灰色 `#888888`
- `sequenceDiagram` / `gantt` / `timeline`：不着色，使用库默认

### → DOCX（default-style.json 字段映射）

- `body.font` = `PingFang SC`，`body.font_en` = `"Gotham"` 降级 `"Helvetica Neue"`
- `headings.h1.color` = `CC0000`
- `headings.h2.color` = `CC0000`
- `headings.h3.color` = `1A1A1A`（左色条用 `CC0000`）
- `table.border_mode` = `"bain"`（表头上方红线）
- `table.rule_color` = `CC0000`（表头上方线色）
- `table.header_rule_bottom_color` = `1A1A1A`
- `table.header_bg_color` = `null`（无背景，纯白）
- 字号、间距参照第 3–4 节
