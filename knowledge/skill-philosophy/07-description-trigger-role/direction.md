# 研究方向：07-description-trigger-role

## 研究命题

description 的职责是触发命中，不是传递操作参数。

## 问题意识

description 同时承担"触发路由"和"说明操作"两种职责，导致作者不知道该写什么——写触发语言嫌它不够清楚，写操作说明又担心影响命中精度。核心矛盾是：**一个字段被两种写作逻辑争夺**。

## 边界与例外

"什么情况下 description 里放操作内容是合理的"本身是需要研究来回答的问题，不预设结论。

明确不在此研究范围内：
- SKILL.md 整体结构设计（body 部分如何组织）
- 触发机制的底层实现（embedding、关键词匹配等技术细节）

## 核心设计疑问

1. **边界在哪里**：什么情况下在 description 里放操作内容是合理的？什么情况下应该彻底分离？
2. **额外信息的必要性**：除触发语言外，哪些额外信息有时必须出现在 description 中？
3. **操作内容的归宿**：如果操作约束不放在 description，它应该写在哪里？

## 研究目标

理解各方取舍，找到可操作的判断标准——什么该放、什么不该放、放不下的去哪里。
