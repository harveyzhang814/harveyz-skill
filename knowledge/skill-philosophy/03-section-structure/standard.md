# Skill 章节结构规范

> 适用于：新建 Skill、Review 现有 Skill 的结构合理性

---

## 核心原则

**章节存在的唯一理由：访问模式不同。**

同一访问模式的内容必须聚合；不同访问模式的内容必须分章。AI 对内容的访问模式只有三种：

| 访问模式 | 含义 | 对应区域 |
|----------|------|----------|
| **顺序执行** | 必须按顺序做，不能跳 | 主体区（步骤） |
| **按需查阅** | 到了某个情境才需要 | 查阅区 |
| **完成时对照** | 做完了检查 / 出问题了诊断 | 完成区 |

---

## 五区模板

```
┌──────────────────────────────────────┐
│  触发区  何时调用这个 Skill          │  所有 Skill 必须有（可在 frontmatter）
├──────────────────────────────────────┤
│  基础区  核心概念 / 初始化           │  建立语义前提或运行环境
├──────────────────────────────────────┤
│  主体区  执行类 = 流程步骤           │  执行类用步骤；参考类用概念体系
│         参考类 = 概念体系            │
├──────────────────────────────────────┤
│  查阅区  按需参考的规则              │  只有部分路径需要时才独立成章
├──────────────────────────────────────┤
│  完成区  质量标准 / 反模式 / 失效模式│  完成时对照，出问题时诊断
└──────────────────────────────────────┘
```

**不是每个区都必须存在：**
- 执行类 Skill → 主体区是流程步骤，可以没有概念体系
- 参考类 Skill → 主体区是概念词汇，可以没有步骤，没有查阅区
- 极短 Skill → 所有区可以内联为隐含分层，不需要显式标题

---

## 创建时：每个章节的存在性检查

新建 Skill 时，每加一个章节前问这三个问题：

1. **这块内容的访问模式是什么？** 顺序执行 / 按需查阅 / 完成时对照
2. **和已有章节的访问模式一样吗？** 一样 → 合并进去，不要新建章节
3. **所有路径都需要，且内容短？** 是 → 内联到相邻内容，不要单独成章

---

## Review 时：章节不应该存在的信号

| 信号 | 处理方式 |
|------|----------|
| 两个章节的访问模式相同 | 合并 |
| 某章节在每条路径都需要且篇幅短 | 内联进相邻内容 |
| 某章节是另一章节的子细节 | 降级为列表项，不独立成章 |
| 某查阅区内容在所有步骤里都要用 | 提升为基础区或内联 |
| 章节标题无法说清"访问时机" | 重命名或合并 |

---

## 两种 Skill 类型的标准结构

### 执行类 Skill

```markdown
## 触发条件
## 初始化（run first）      ← 基础区，每次必跑
## 流程（STRICT）           ← 主体区，有相位编号
### 阶段 1 ...
### 阶段 2 ...
## [规则名]                 ← 查阅区，按需跳转
## 完成标准 / 反模式        ← 完成区
```

### 参考类 Skill

```markdown
## 核心原则（无标题或直接正文）  ← 基础区
## [概念 A]                      ← 主体区，词汇体系
## [概念 B]
## 何时 [做某个决策]             ← 仍是主体区，决策规则
## 失效模式                      ← 完成区，诊断用
```

---

## 可直接复制的起始模板

以下两段模板可以直接复制到新 Skill 的 `SKILL.md` 中作为起点。

**使用时需要替换的占位符：**
- `{Skill 的核心动作}` — 这个 Skill 在做什么（一句话）
- `{触发词}` — 用户说什么话时触发
- `{阶段 N 名称}` — 流程步骤的实际名称
- `{规则名}` — 查阅区的章节标题
- `{概念 N}` — 参考类的核心概念名称

**使用时需要删除的内容：**
- `← 注释` 行（这些是给人看的结构说明，不是 Skill 内容）
- 不适用于这个 Skill 类型的整个区块

---

### 执行类起始模板

```markdown
---
name: {skill-name}
description: Use when {触发词}. {补充触发场景}.
---

{Skill 的核心动作}。触发条件：{触发词}。

---

## 初始化（run first）

{每次运行前必须做的准备工作，例如：读取配置、检查环境、确认前置条件}

---

## 流程（STRICT — 不可跳过或合并步骤）

### 阶段 1：{阶段 1 名称}

{这个阶段做什么，完成标准是什么}

完成标准：{可检验的完成条件，例如"所有文件已读取"而非"完成了解"}

### 阶段 2：{阶段 2 名称}

{这个阶段做什么，完成标准是什么}

完成标准：{可检验的完成条件}

---

## {规则名}

{只在特定情境下才查阅的规则，例如"如何格式化输出"、"遇到歧义时怎么处理"}

---

## 完成标准

- {可验证的产出物 1}
- {可验证的产出物 2}

## 反模式

| 错误行为 | 正确做法 |
|----------|----------|
| {常见错误 1} | {应该怎么做} |
| {常见错误 2} | {应该怎么做} |
```

---

### 参考类起始模板

```markdown
---
name: {skill-name}
description: Reference for {主题} — {一句话说明这个参考手册覆盖什么}.
disable-model-invocation: true
---

{核心原则：这个领域的底层逻辑是什么，一到两句话}

---

## {概念 1}

{定义、规则、注意事项都在这里，不要散落到其他章节}

---

## {概念 2}

{同上}

---

## 何时{做某个关键决策}

{执行某个判断时的决策规则，例如"何时拆分"、"何时内联"}

- **{情况 A}**：{做法}
- **{情况 B}**：{做法}

---

## 失效模式

用于诊断：当这个 Skill 的使用者出现问题时，对照以下模式。

- **{失效模式名称}** — {描述症状和原因}。修复：{怎么解决}。
- **{失效模式名称}** — {描述症状和原因}。修复：{怎么解决}。
```
