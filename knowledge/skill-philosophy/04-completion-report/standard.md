# Skill 完成回报格式规范

> 适用于：任何需要向用户展示任务完成结果的 Skill

---

## 卡片壳

主 agent 完成任务后，向用户输出一张卡片：

```
── {状态} ──────────────────────────
{字段名}  {字段值}
...
────────────────────────────────────
```

**状态值（固定四种）：**

| 状态 | 含义 |
|------|------|
| 完成 | 全部步骤成功 |
| 失败 | 任务未能完成 |
| 部分完成 | 主要步骤成功，次要步骤失败 |
| 已跳过 | 任务未执行（如 dedup 命中） |

---

## 字段规则

- 字段名与值之间用两个空格对齐
- 字段顺序由各 skill 自行定义
- `原因` 字段：失败 / 部分完成 / 已跳过时**必须**出现，一句话说明
- 路径类字段：仅在有输出文件时出现；部分完成时注明实际保存位置
- 各 skill 可自由扩展字段，但不得改变卡片壳结构

---

## 批量模式

每项完成立即输出一张卡片（不等全部完成）。所有项完成后追加汇总行：

```
共 N 篇 | 完成 X  失败 Y  跳过 Z
```
