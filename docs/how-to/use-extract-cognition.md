# 如何使用 extract-cognition：从一篇文章学到可迁移的思维动作

本指南讲**怎么用**这个 skill、产出哪些文件、该读哪个、怎么把结果用起来。想理解它背后的哲学（为什么这么设计），见 [explanation/cognitive-signature-philosophy.md](../explanation/cognitive-signature-philosophy.md)。

---

## 什么时候用

当你想从一篇文章里**学到"作者是怎么思考/怎么写的"**，而不只是知道它讲了什么。典型触发：

- "这篇论证手法我能学到什么"
- "拆解这篇背后的思路，好让我自己也能用"
- "教我像这位作者一样思考/写作"
- "这篇用了哪些可复用的认知动作"

不适用：纯总结内容、跨多篇做作者画像（本 skill 只处理单篇）。

## 前置条件

- 一个**本地** `.md` / `.txt` 文件，全文（非片段），约 ≥300 字。
- 不抓取 URL、不解析 PDF。
- 文本里要有论述成分；纯数据/表格/代码会被拒绝（无认知痕迹可抽）。

## 第一步：决定模式（A 还是 B）

| 你想要的 | 用哪个模式 | 要不要基线 |
|---|---|---|
| 学这套写作/思维手法、自己也能用 | **模式 A** | 不需要 |
| 还想知道"这些招是这位作者的特质，还是体裁通用套路" | **模式 B** | 需要 2–3 篇*同类型、不同作者*的文章作基线 |

**绝大多数"学习"目的用模式 A 就够。** 模式 B 多做的是归因到作者本人——那是研究作者，不是学手法。

## 第二步：运行

触发 skill，给出文章路径即可。首次运行会问你产出保存目录（默认 `~/Documents/cognition`）。

可选的分段运行（`--pass`），适合长文或想分步看：

| 参数 | 跑什么 | 产出 |
|---|---|---|
| `--pass 1` | 表层精读 + 逻辑图谱 | `1-evidence.md` |
| `--pass 2` | 描述层签名卡 | `2-signature.md` |
| `--pass 3` | 认知动作手册 | `3-playbook.md` |
| `--pass 4` | 归因层（仅模式 B，需重新提供原文+基线） | `4-attribution.md` |
| 无参数 | 一路跑到手册（模式 B 再加归因层） | 全部 |

模式 B 想跑 `--pass 4` 时，要**重新提供原文路径与基线路径**——留出验证要读原文留出段、Agenticity 要读基线，光有签名卡不够。

## 第三步：该读哪个文件

产出在 `<输出目录>/<文章slug>/` 下。**关键：阅读次序和生成次序相反——只通读一个文件，其余按需下钻。**

| 文件 | 角色 | 怎么用 |
|---|---|---|
| **`3-playbook.md`** ★ | 认知动作手册（头牌） | **通读 + 动手练。唯一为"学"而写的文件。** |
| `2-signature.md` | 证据/审计层：签名卡 | **查证用，不通读。** 怀疑某招时，顺手册里「回指证据 → 签名卡 #N」翻到对应卡看逐字实例 |
| `1-evidence.md` | 残余日志 + 逻辑图谱 | **取证底料，基本不打开。** 想追到最原始的逐字定位才下钻 |
| `4-attribution.md` | 归因层（仅模式 B） | 只在你想研究"这是不是作者本人特质"时读；纯学习目的可忽略 |

一句话：`3-playbook.md` 是课本，`2-signature.md` 是参考文献，`1-evidence.md` 是原始档案，`4-attribution.md` 是关于作者的旁支论文。

## 第四步：怎么把 `3-playbook.md` 用起来

不要从头背到尾。按这个次序：

1. **先吃透「一、发生器」**——那 1–2 条根性镜头是"学会就能自己再生招"的那层。只背具体招，换个题目就忘；记住根性，才能在自己的题目上现场长出新招。
2. **挑你当前用得上的招**。每张卡有 `该学还是该防` 一栏：
   - 标 `✓` 的是可直接纳入工具箱的 craft；
   - 标 `✗` 的是该识破、防御的话术；
   - 标 `~` 的是双刃，看场合。
3. **做「三、迁移练习」**——拿你自己要写的东西套一遍。**学习真正发生在这一步**，前面都只是输入。

## 排错

| 现象 | 原因 / 处理 |
|---|---|
| skill 拒绝执行 | 文本太短/是片段/无论述成分 → 换完整文章 |
| 想归因却没产 `4-attribution.md` | 没提供基线 → 自动降级模式 A（手册照常给）；要归因就补 2–3 篇同类基线 |
| 某条招看着像"鸡汤" | 翻它的 `回指证据 → 签名卡`，看逐字实例是否站得住；站不住就是该被 F6 锁拦下的漏网，反馈即可 |
