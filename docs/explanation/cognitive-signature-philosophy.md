# extract-cognition 背后的哲学：把文章当作一颗脑子的痕迹

本文讲**为什么**这个 skill 是现在这个样子——它的世界观、认识论立场、以及几个关键设计取舍。想直接上手用，见 [how-to/use-extract-cognition.md](../how-to/use-extract-cognition.md)。方法论原文与学术根基在 `docs/reference/cognitive-signature/`。

---

## 一、核心隐喻：文章是认知的痕迹

一篇论说文是一颗脑子留下的痕迹。作者怎么切问题、把什么当不证自明、在哪里用隐喻顶替论证、在哪里突然自信——这些都印在文字里。本 skill 的根本动作是**逆向**：从痕迹反推出那套产生它的思维方式。

但有一条铁律：**我们重建的是"隐含作者"，不是真人。** "隐含作者"是这篇文本所投射出的认知形象。真人 Dan Koe 是谁、私下怎么想，我们无从、也不声称知道。产物永远是**带校准置信度的假设**，不是对某个人心智的确证。把假设写成事实，是这套方法第一个要防的错误。

## 二、两种目的，决定了两种 skill

同一批观察，可以服务两个完全不同的目的——这是本 skill 从 0.1.0 到 0.2.0 重构的根因。

- **法证式（forensic）**：目的是产出一个站得住、防过度归因的"作者签名"。所有机制都为**压制假阳性**服务——别声称一个其实不属于作者的模式。
- **教学式（learning）**：目的是把**可迁移、能复用的认知动作**捞出来，讲到你能自己用。

换了目的，几样东西的好坏就翻转了：

| 维度 | 法证式（旧） | 教学式（新） |
|---|---|---|
| 怕的错误 | 假阳性（误指一个模式） | 假阴性（漏掉一个可学的招） |
| 分析单位 | 作者的**独特**之处 | **有效且可迁移**的动作（独特与否不重要） |
| 读取方向 | 反向：有什么 / 缺什么 | 正向：怎么再造出来（配方） |
| 判断轴 | 归因置信度 | 该招何时灵 / 何时砸 |

**最反直觉的一点**：法证式在猎"独特性"，而独特恰恰是最不可迁移的（怪癖学不走）。它的注意力天生背对最该学的料——那些**扎实、常规、有效**的写作真功夫。教学式刻意把这个代价翻转过来：常规但有效的招照样保留。

## 三、为什么"先学发生器，而不是背一串招"

一套思维方式是个**系统**，表面的十几招都是从一两条根性长出来的。这一两条根性，就是作者默认不证自明、从不拿出来论证的前提（在方法里叫 **warrant**）。

所以手册开篇先给「发生器」（那 1–2 条根性镜头），再把每招标注为"从哪条根性长出"。理由很简单：

> 学会镜头，换了新题目你能自己再生出招；只背招，换题就忘。

这也是为什么 **warrant 是信息量最大的一栏**。文章里**写出来的论证**是作者愿意给你看的；**warrant 是他自己都没意识到、当成地基的东西**。认知签名的指纹，长在没说出口的地方。

## 四、签名怎么来：负向定义，反清单扫描

一个诱人的错误是：拿着"系统思维？第一性原理？"这类预设标签去文本里找对应。确认偏误下你一定"找得到"。这是本方法头号失败模式（F1 清单扫描）。

正确的做法是**负向定义**——让签名从两类"不合群"里自己冒出来：

1. **残余**：隐喻、对冲情态、反常选词、离题旁白——任何让你"咦"的地方。
2. **结构空隙**：逻辑图里缺失的边、承重却零支撑的节点、自信的不对称。

签名是"残余 × 空隙"的交叉，不是清单的命中。这保证了产物**接地**——每条都钉在文本的具体位置上。

## 五、防鸡汤的硬锁

教学层最大的风险，是滑成漂亮但空洞的鸡汤。两道二元闸门挡住它：

- **F6 回指锚**：每张认知动作卡**必须**指向一张签名卡的逐字实例。钉不到证据的招，不许写出来。
- **F7 可迁移**：招名和步骤里若出现本文专有名词/具体话题，判不可迁移，必须改写成内容无关的通用操作——迁移练习能换题套用才算过。

这两道锁让"学习层"建在"证据层"之上，而不是漂在空中。教学没有以牺牲严谨为代价。

## 六、该学，还是该防

同一篇文章里混着两种东西：值得纳入工具箱的真功夫，和该警惕的话术。一个诚实的手册必须区分，所以每招都带一行 `该学还是该防`：

- `✓` 可直接学的 craft（如先示弱建立信任、把抽象兑成可执行清单）；
- `✗` 该识破的话术（如用重定义顶替论证、把失败预先归因给读者使主张不可证伪）；
- `~` 双刃（如承重隐喻——是高效把手，也可能是论证薄弱处的遮盖）。

学习一个招，不只是会用它，还包括知道它何时有力、何时是陷阱。这种"判别"才让招真正可用。

> 注：0.2.0 选择了 **generative 为主**——重点是"我能怎么用"，防御性判别退为这一行附注。代价是：若你的目的是纯防御（看穿被操纵），手册对"纯破绽类"洞察会相对收敛。这是有意的方向选择，不是缺陷。

## 七、为什么"无基线就不归因"

要说"这是**作者本人**的稳定特质"（而不只是"本文这么写了"），你必须排除一个对手解释：**这只是体裁的通用套路**。self-help 长文本来就爱用重定义、爱堆权威。

N=1（单篇、无对照）时，"作者信号"和"体裁信号"在数学上不可分离。所以有一条红线：**没有体裁基线，就不输出任何归因层签名**。无基线的归因是已知的失败模式，不是可接受的近似。

这条红线只约束归因层（`4-attribution.md`）。**学习层不受它限制**——因为学一个招怎么用，根本不需要先知道它是谁的。体裁标配的招，照样值得学。

## 八、一句话收束

法证版回答"**这篇文章在干嘛**"；学习版回答"**我怎么也能这么写、以及哪几招我该警惕**"。前者把文本当标本诊断，后者把读者当主体装备。本 skill 选择了后者，同时用证据锚定和归因红线，守住"不把假设当事实、不为产出降标准"的底线。
