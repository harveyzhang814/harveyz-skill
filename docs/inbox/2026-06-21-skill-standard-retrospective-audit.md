## 四、关于标准本身的诊断

这次审计同时**反向评估了标准**：若 25/30 个 skill 都违反某条哲学，可能不是 skill 的问题，而是哲学的问题。

### 标准过严的怀疑（Φ3）

**Φ3 "必须有不在范围内节" 在 30 个 skill 中 25 个违反。** 三种解释：

1. **标准过严**——很多 skill（如 extract-vision、draw-diagram）功能边界天然清晰，强加"不在范围内"是过度规范化
2. **codebase 系统性失误**——所有作者都漏写了这一节
3. **标准措辞不清**——可能"不在范围内"应该是 SHOULD 而非 MUST，或允许其他表达

**建议：** 降为 SHOULD 或加豁免条款"功能单一无边界混淆风险时可省略"。

### 标准措辞模糊的怀疑（Φ10）

**Φ10 47% 合规** 大多是因为 SKILL.md 用了 ✓ 符号但没**显式列出符号约定表**。这不是真违规，是文档形式问题。

**建议：** Φ10 应放宽——"输出中实际使用统一符号即可，无需显式列约定表"。

### 标准切中要害的（Φ1、Φ12）

**Φ1 / Φ12 高违规说明真实风险存在：**
- 半数破坏性操作没有确认机制 → 真实安全隐患
- 处理用户原始输入的 skill 多数没有清洁化 → 真实注入面

这两条**不应放宽**，反而应**优先修复违规 skill**。

## 五、结构性发现

### A. 安全相关哲学的两极分化

| 哲学 | 合规率 | 性质 |
|------|------|------|
| Φ2 故障保留 | 97% ✓ | 几乎所有 skill 都做了前置检查 |
| Φ9 不越权 | 84% ✓ | 多数遵守 |
| **Φ1 可回退** | **48% ✗** | **半数没做** |
| **Φ12 输入清洁化** | **25% ✗** | **多数没做** |

→ "防失败"做得好（Φ2/Φ9），"防误操作 + 防注入"做得差（Φ1/Φ12）。前者是**被动安全**，后者是**主动安全**。codebase 在主动安全上有系统性短板。

### B. 形式相关哲学的反差

| 哲学 | 合规率 |
|------|------|
| Φ18 形式即接口 (frontmatter + 命名) | 97% |
| Φ3 边界穷举 ("不在范围内"节) | 17% |
| Φ10 输出符号 | 47% |

→ 机器可校验的形式（frontmatter）几乎全合规；语义形式（哲学性的节、符号约定）大量违规。**说明形式规范要靠工具强制，不能靠作者自觉。**

### C. 任务分阶段相关哲学全合规

Φ4 信息按需、Φ5 配置就地、Φ8 可组合、Φ11 关注点分离、Φ13 自我进化、Φ15 任务分阶段——全部 100%。

→ 当 skill 涉及这些哲学时，作者都本能地做对了。这些哲学**反映的是 skill 的自然结构**，不是外加约束。

## 六、给标准的修订建议

按"应保留 / 应放宽 / 应强化"分类：

### 应保留（核心安全）

- Φ1 可回退（48%）
- Φ12 输入清洁化（25%）
- Φ16 证据可溯源（56%，分析类 skill）

**不放宽，但应：** 把违规 skill 列入"待修复"，逐个补齐。

### 应放宽（标准过严）

- **Φ3** "必须有不在范围内节"：MUST → SHOULD，加豁免条款
- **Φ10** "必须显式列符号约定表"：MUST → SHOULD，实际使用即可

### 应强化（高合规率但仍有漏网）

- Φ18 形式即接口 97% 已经很好，1 处违规可机器修复

### 新加规则（标准盲区）

审计发现一个标准未覆盖的模式：**多个 skill 把 "应该写但没写不在范围内" 当作默认行为**。这反过来暗示标准需要"允许省略 + 必须显式说明为何省略"的豁免机制。

## 七、自评

### 评估方法的局限

- **单 LLM judge**：subagent 的 applies/complies 判断本身有主观性，无 inter-rater agreement
- **触发判断不一致**：同一种模式（如"接收文件路径"）在不同 skill 上被判 Φ12 触发与否不一致
- **过细 vs 过粗**：标准的"必做"有时是多项组合，subagent 可能只看主项就判合规

### 数据可信度评级

| 哲学 | 数据可信度 |
|------|---------|
| Φ18（机器可检） | 高 |
| Φ2/Φ9（明确模式） | 中高 |
| Φ1/Φ3/Φ10（半语义） | 中 |
| Φ11/Φ14（深语义） | 低 |

复杂语义判断（Φ11 关注点分离）的合规率 100% 可能是 subagent 标准过宽，而非真合规。
