## 1. 目标与定位

从单篇文章中提取作者的**推理逻辑**与**思维方式**，分三步递进产出：先还原论证的逻辑结构（图谱），再从结构中涌现式抽象出思维方式全集，最后以用户指出的"吸引点"为导航锚做重点深入。

这是一个**独立的分析 skill**——本设计不考虑它与"元框架提取 skill"的衔接关系；后续若需要作为元框架提取的前置层，再单独处理。

**核心原则**：
- **结构先于抽象**：先有逻辑图谱（客观还原），再有思维方式（解释性抽象），不跳步。
- **涌现而非套框架**：思维方式从文本中归纳得出，不预设固定维度（数据/分析/推理只是举例，不是纲领），不限于演绎一种。
- **导航锚而非筛选器**：用户的吸引点决定"先讲什么、深挖哪条切入口"，而非决定"只保留什么"。文中所有推理链与思维方式在最终产出中都完整保留。

---

## 2. 输入与配置

### 输入
- 本地 `.md` / `.txt` 文件路径（绝对路径或 `~` 开头）。
- 仅处理本地文本文件；不抓取 URL、不解析 PDF（如需可后续扩展）。

### 配置（仿 learn-paper）
- `ConfigPath: ~/.hskill/extract-reasoning/config.json`
- 字段 `output_dir`，默认 `~/Documents/reasoning`
- 首次运行若配置不存在则询问用户保存目录，路径写入时用 `$HOME` 展开，不写字面量 `~`。

### 产出目录
```
<output_dir>/<slug>/
├── 1-logic-map.md   ← 步骤一：逻辑图谱
├── 2-modes.md       ← 步骤二：思维方式全集
└── 3-anchored.md    ← 步骤三：锚定深入
```
slug 由文件名生成（去扩展名、转小写、非字母数字/汉字替换为 `-`、合并连字符）。

## 3. 参数与流程控制

仿 learn-paper 的 `--pass` 机制：

| 参数 | 行为 | 依赖 |
|------|------|------|
| `--pass 1` | 只执行步骤一 | 无 |
| `--pass 2` | 只执行步骤二 | `1-logic-map.md` 已存在 |
| `--pass 3` | 只执行步骤三 | `2-modes.md` 已存在 |
| 无参数 | 执行全部三步 | — |

**锚点（吸引点）输入**：可选。
- 触发消息中已给出锚点 → 三步连续跑到底。
- 未给锚点 → 步骤二产出后**暂停**，把思维方式清单呈现给用户，请其指认锚点，再跑步骤三。

---

## 4. 步骤一 — 逻辑图谱（结构层）→ `1-logic-map.md`

**读取**：完整读取输入文本文件。

**任务**：把文章的论证还原为显式逻辑结构。
- **节点类型**：前提 / 证据 / 中间结论 / 最终结论。
- **边**：推理动作，边上标注推理类型（如"归纳""演绎""类比""反证""因果"）。
- 文中若有多条交织链条，统一纳入一张 mermaid flowchart（过于庞大时可拆分图，但需说明分图依据）。
- **逐链文字走读**：每条主推理链一句话说清"从什么、靠什么、推到什么"。

**产出模板**：
```markdown
# {文章标题} — 逻辑图谱

**来源文件**: <input_path>
**分析日期**: YYYY-MM-DD

---

## 逻辑图谱

​```mermaid
flowchart TD
  P1[前提: ...] -->|归纳| C1[中间结论: ...]
  E1[证据: ...] -->|支撑| C1
  C1 -->|演绎| CONC[最终结论: ...]
​```

## 逐链走读

1. **链 A**：从 {前提/证据} 出发，靠 {推理动作} 推出 {结论}。
2. **链 B**：...

## 结构说明

（多链如何交织、主链与支线关系、是否存在并列或递进结构）
```

写完告知：`✓ 步骤一完成 → <output_dir>/<slug>/1-logic-map.md`

---

## 5. 步骤二 — 思维方式全集（抽象层）→ `2-modes.md`

**读取**：只读 `1-logic-map.md`，不重读原文。

**任务**：从逻辑图谱中**涌现式**归纳作者用到的思维方式。
- **不预设固定维度**，不强行套"数据/分析/推理"三类；类别由文本实际内容决定。
- **不限于演绎/推导**一种——可能含归纳、类比、反事实、第一性原理、系统思维、辩证、框架重置、数据处置策略、问题重构等，取决于文章实际用了什么。
- 每种思维方式记录：`名称` + `在本文中的含义` + `出现位置（指向图谱节点/边或链编号）` + `原文关键句（保留原语言）`。

**收尾模式归类表**：思维方式 × 出现链条 × 频次。

**产出模板**：
```markdown
# {文章标题} — 思维方式全集

**来源文件**: <input_path>
**分析日期**: YYYY-MM-DD
**依据**: 1-logic-map.md

---

## 思维方式清单

### {思维方式名称}
- **在本文中的含义**: ...
- **出现位置**: 链 A 的 P1→C1 / 节点 ...
- **原文关键句**: "..."

（逐条列出所有识别到的思维方式）

## 模式归类表

| 思维方式 | 出现链条 | 频次 |
|----------|----------|------|
| ... | A, C | 3 |
```

**若无锚点**：写完后呈现清单并暂停，提示用户从中指认吸引点，再进入步骤三。
写完告知：`✓ 步骤二完成 → <output_dir>/<slug>/2-modes.md`

---

## 6. 步骤三 — 锚定深入（应用层）→ `3-anchored.md`

**读取**：只读 `2-modes.md` + 用户锚点；不重读原文与图谱。

**锚定语义**：
- 锚点模式**优先重点阐述**，并以它为切入口**深入分析整篇文章**（锚点是导航起点，分析对象仍是文章整体）。
- **其余思维方式/推理链完整保留**在输出中，仅次序靠后、篇幅次之。
- 若文中存在多条推理链，锚点所涉链条优先展开。

**产出模板**：
```markdown
# {文章标题} — 锚定深入

**来源文件**: <input_path>
**分析日期**: YYYY-MM-DD
**锚点**: {用户指出的吸引点}
**依据**: 2-modes.md

---

## 锚点深入：{锚点模式}

（重点阐述该模式：它如何贯穿文章、为何有效、以它为切入口对全文的再解读）

## 其余思维方式（保留）

（逐条简述步骤二中其余模式，保持完整性，不删减）

## 小结

（锚点视角下，这篇文章的认知方法对读者的可复用价值）
```

写完总结三文件树。
