## 1. 目标与定位

把一篇文章当作认知的痕迹，反推出**隐含作者**的思维方式，输出为**带证据、带置信度、带对立解释**的结构化"认知签名"。

这是一个**独立的分析 skill**——本设计不考虑它与"元框架提取 skill"的衔接。

本 skill 不是从零设计，而是把已完成的底层研究**编译落地**。四份研究文档（现存 `docs/inbox/`）的角色：

| 文档 | 角色 | 落地去向 |
|---|---|---|
| ④ cognitive-signature-framework.md（可执行框架） | **母本** | 编译为 `SKILL.md` |
| ① article-analysis-methods.md（方法工具箱 / Layer 1） | **reference** | `references/article-analysis-methods.md` |
| ② cognitive-signature-methodology-v2.md（管线 v2，人读） | 设计依据 | 移入 `docs/reference/cognitive-signature/` |
| ③ methodology-academic-grounding.md（学术根基） | 边界依据 | 移入 `docs/reference/cognitive-signature/` |

**操作信条（写进 SKILL.md，执行前内化）：**
- 产出的是**关于隐含作者的假设**，不是对真人心智的确证；永不把假设写成事实。
- 默认怀疑自己：每条签名默认"未证实"，由证据和闸门推向"更可信"。
- 宁可少声称而站得住，不可多声称而虚高。

## 2. 输入与配置

### 输入
- **主文章**：本地 `.md` / `.txt` 文件路径（待分析全文，非片段）。
- **体裁基线（可选，模式 B 必需）**：2–3 篇*同类型、同场域、非同作者*的本地文本文件路径，仅作"体裁尺子"，用完即弃。

### 硬停机条件（满足任一则拒绝执行并说明）
- 主文章少于约 300 字或明显是片段 → 样本不足。
- 纯数据/表格/代码、无论述成分 → 无认知痕迹可抽。

### 配置（仿 learn-paper）
- `ConfigPath: ~/.hskill/extract-cognition/config.json`，字段 `output_dir`，默认 `~/Documents/cognition`。
- 首次运行若无配置则询问保存目录；写入用 `$HOME` 展开，不写字面量 `~`。

### 产出目录
slug 由主文章文件名生成（去扩展名、转小写、非字母数字/汉字替换为 `-`、合并连字符）。
```
<output_dir>/<slug>/
├── 1-evidence.md        ← 阶段 1–3：残余日志 + 逻辑图谱(+补链日志) + 结构信号
├── 2-descriptive.md     ← 阶段 4(+6/7)：描述层签名卡 + 描述画像 + 上限声明 + 自审
└── 3-attribution.md     ← 阶段 5–7：归因层签名卡 + 归因画像 + 自审（仅模式 B）
```

## 3. 模式判定（执行前必须确定，不可跳过）

| 条件 | 模式 |
|---|---|
| 无体裁基线，或用户只想分析本文 | **模式 A：仅描述层** |
| 有 2–3 篇体裁基线，且用户想归因到作者 | **模式 B：描述层 + 归因层** |
| 想归因但无基线 | 先请求基线；用户拒绝则**降级模式 A**，并明确告知"无基线无法归因到作者" |

**绝对红线：无体裁基线时禁止输出任何归因层签名。** N=1 时"作者信号"与"体裁信号"不可分离，无基线的归因是已知失败模式。

交互：触发时若用户已提供基线路径并表示要归因 → 直接走模式 B；否则进入模式判定对话（一次问清是否要归因、是否有基线）。

## 5. 执行管线（SKILL.md 主体，编译自 ④）

SKILL.md 完整保留 ④ 的 7 阶段确定性管线（每阶段：进入 / 动作 / 二元判据 / 产出），仅做"落地适配"——把"输出签名卡"改为"写入上述三个文件"。各阶段要点：

**状态产物（全程显式维护）**：RESIDUE_LOG（残余日志，逐字 span + 定位）、WARRANT_LOG（补链日志，未明说前提）、SIGNATURE_CARD（签名卡，schema 见第 6 节）。

- **阶段 1 表层精读（先别拆，禁止建图）** → RESIDUE_LOG：隐喻/框架、对冲情态、局部连接词、价值词、反常选词/旁白。每条须逐字定位。
- **阶段 2 建逻辑图谱** → 判型（四问 + 查 `references/article-analysis-methods.md` 选拆解法）→ 建图（节点+带含义的边，mermaid）→ 每条暗连写入 WARRANT_LOG。
- **阶段 3 结构判读** → 对图本身读：形状（深/宽、交叉/纯树）、承重节点、缺失的边、不对称。各指向图中具体位置。
  - 阶段 1–3 产出汇入 `1-evidence.md`。
- **阶段 4 两源交叉 → 描述层候选** → 残余分诊（剔疲劳/凑字/编辑痕迹）→ 残余×空隙交叉 → 描述层签名卡（"本文实现了 X" + ≥1 逐字实例）。组装+置信度+自审后写入 `2-descriptive.md`。**模式 A 到此交付。**
- **阶段 5 归因闸门流水线（仅模式 B）** → 对每条描述候选独立走：Patternicity（≥2 逐字实例）→ Agenticity（基线减除后仍突出）→ 登记仍存活的对立解释 → **独立对抗 pass**（结构上独立地找最廉价的非认知解释）→ **留出验证**（预留 20–30% 文本，用签名预测留出段属性再核对）。任一硬闸门 FAIL 即丢弃该条。
- **阶段 6 组装 + 置信度** → 描述层：高/中/低（实例数+替代读法）；归因层强度 = 实例数 − 仍存活对立解释数，叠加留出结果；**归因层置信度必须明确低于其描述层对应项**。
- **阶段 7 交付前自审** → 过 8 项自审清单（无基线无归因、每条有足量逐字实例、假设未写成事实、跑过对抗+留出、列了对立解释、无清单扫描痕迹、附上限声明等），任一不过返工。阶段 5–7 产出汇入 `3-attribution.md`。

**失败模式监控（持续）**：F1 模式强加/清单扫描、F2 流畅过度归因、F3 体裁信号冒充签名、F4 置信度膨胀、F5 残余过读。命中即按 ④ 的纠正动作处理。

## 6. 输出契约（签名卡 schema，取自 ④）

每条签名一张两层卡，描述层可单独交付，归因层仅模式 B 填：
```
签名 #N：[一句话命名这个特征操作]

── 描述层(关于本文) ──
  本文实现:     [客观描述本文中的这个认知操作]
  证据实例:     [≥1 处逐字片段 + 定位]
  来源:         表层颗粒 / 结构骨架 / 残余 / 空隙
  描述层置信度:  高 / 中 / 低

── 归因层(关于隐含作者；模式 A 留空) ──
  归因主张:     [指向隐含作者的什么思维方式]
  体裁基线对照:  [对照文本；扣除了什么体裁信号]
  Patternicity:  PASS(≥2 实例) / FAIL
  Agenticity:    PASS(基线减除后仍突出) / FAIL
  对立解释:      [仍说得通的非认知读法，逐条；未排除数 = N]
  对抗 pass:     [最强反驳 + 为何未推翻]
  留出验证:      [预测了留出段什么属性；是否命中]
  归因层置信度:  强 / 暂定(明确低于描述层)
  关联 Warrant:  [背后那个作者默认不论证的前提]
```

**整体画像**：描述层卡片汇成"本文实现了什么认知风格"；归因层卡片汇成关于隐含作者的假设集。**未明说的 Warrant 是信息量最大的一栏，须在画像中突出。**

**上限声明（每次输出必附）**：
> 以上为对**隐含作者**（文本所投射的认知形象）的分析。基于单篇，产物为**带校准置信度的假设**而非确证；"是否为作者稳定特质"需跨文本，本框架不作此声称。

## 7. 停机 / 降级 / 求助

- 想归因但无基线 → 自动降级模式 A，告知用户。
- 命中硬停机条件 → 拒绝执行。
- 基线是否合适、体裁难判、文本含大量他人引文难分作者声音 → 向用户澄清而非猜测。
- **空结果是合法结果**：所有候选都未过闸门 → 如实交付"未发现可证实的归因层签名，仅有描述层观察"，不为产出而降标准。
