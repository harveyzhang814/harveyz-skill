# init-goal Skill Design

## 概述

`init-goal` 是一个对话式引导 skill，帮助用户为 `/loop` 命令生成结构化的初始化 prompt。
通过分步问答收集目标、执行步骤、评估指标、约束和退出条件，最终生成一个静态的 `prompt.md` 文件。
`/loop` 以此文件为驱动，过程记录在独立的 `log.md` 中，结束时生成 `summary.md` 供回顾。

---

## 触发场景

- 用户输入 `/init-goal`
- 用户说"帮我初始化一个 loop 目标"、"设定一个 GOal"、"我要用 loop 完成 X"

---

## 文件架构

```
~/.hskill/init-goal/<goal-slug>/
├── prompt.md     # 初始 prompt（静态，用户主动编辑才变）
├── log.md        # 过程文档（每轮 loop 末尾追加）
└── summary.md    # 总结文档（loop 退出时生成）
```

`<goal-slug>` 由 skill 根据目标文本自动生成（kebab-case，如 `fix-auth-tests`）。

---

## 对话流程

### Step 1 — 模版选择

展示内置模版菜单，用户选择一个或跳过（从零开始）：

| # | 模版名 | 适用场景 |
|---|---|---|
| 1 | Fix Until Green | 持续修 bug 直到测试全通过 |
| 2 | Research Loop | 反复搜索直到信息足够 |
| 3 | Refine Until Satisfied | 迭代优化某个输出 |
| 4 | Monitor & React | 持续监控状态并响应变化 |
| 5 | Explore & Map | 系统性探索未知领域/代码库 |

选择模版后预填对应字段，后续步骤只追问空缺或需定制的字段。
不选模版则走完整对话，每步均需用户填写。

### Step 2 — 目标

问："你想达成什么？成功是什么样子？"

模版有默认描述时展示并询问是否修改。用户的回答直接作为 `prompt.md` 的 `## GOal` 内容。

### Step 3 — 每轮执行内容

问："每次 loop 应该做什么动作？"

引导用户描述每轮的具体步骤（可以是列表）。模版有默认步骤时展示供用户接受或覆盖。

### Step 4 — 评估指标

问："每轮结束后如何判断进展？（打分 / 检查清单 / 比对结果…）"

用户描述的评估方式写入 `## 评估` 段落。

### Step 5 — 约束条件

问："有什么限制？（最多轮数、不能改动的文件、必须保留的行为…）"

可回答"无"跳过，写入 `## 约束` 段落。

### Step 6 — 退出条件

分两个子问题：
1. "达到什么状态可以停止？"（明确条件）
2. "如果 Claude 无法判断是否达到目标，应该怎么做？"（兜底逻辑，如"连续 2 轮无实质进展则停止并汇报"）

### Step 7 — 汇总确认

展示所有字段摘要，用户确认或指出需要修改的字段，修改后再次确认。

### Step 8 — 生成文件

- 根据目标文本生成 `<goal-slug>`
- 写入 `~/.hskill/init-goal/<goal-slug>/prompt.md`
- 输出启动命令提示（不含 interval，用户自选）

## 内置模版详情

### 1. Fix Until Green

| 字段 | 默认值 |
|---|---|
| 目标 | 持续修复代码问题，直到所有测试通过 |
| 每轮执行 | 1. 运行测试套件；2. 分析失败用例；3. 定位根因；4. 尝试修复；5. 重新运行验证 |
| 评估指标 | 测试通过率（通过数 / 总数），记录本轮净增通过数 |
| 约束 | 不修改测试文件本身；每轮最多修改 3 个文件 |
| 退出条件 | 明确：全部测试通过；兜底：连续 2 轮通过率无变化则停止并报告卡点 |

### 2. Research Loop

| 字段 | 默认值 |
|---|---|
| 目标 | 持续搜索和整理信息，直到对目标主题有足够深度的理解 |
| 每轮执行 | 1. 根据当前空白选择下一个搜索方向；2. 执行搜索；3. 摘要新发现；4. 记录到 log |
| 评估指标 | 本轮新增有效信息条数；主要问题是否已有答案 |
| 约束 | 每轮搜索不超过 5 个查询；不重复已搜索过的关键词组合 |
| 退出条件 | 明确：所有预设问题均已有答案；兜底：连续 2 轮无新发现则停止 |

### 3. Refine Until Satisfied

| 字段 | 默认值 |
|---|---|
| 目标 | 反复优化指定输出，直到质量达到满意标准 |
| 每轮执行 | 1. 评审当前版本；2. 列出 3 个最重要的改进点；3. 执行改进；4. 自评分（1-10） |
| 评估指标 | 自评分，记录分数变化趋势 |
| 约束 | 每轮只改进不超过 3 处；不推翻上一轮已确认的改动 |
| 退出条件 | 明确：自评分 ≥ 8；兜底：连续 2 轮分数不再提升则停止并说明瓶颈 |

### 4. Monitor & React

| 字段 | 默认值 |
|---|---|
| 目标 | 持续监控指定状态，发现变化时执行响应动作 |
| 每轮执行 | 1. 检查目标状态；2. 与上轮状态对比；3. 若有变化则执行预定响应动作；4. 记录状态快照 |
| 评估指标 | 状态是否稳定；本轮是否触发响应动作 |
| 约束 | 响应动作仅限预定范围，不做范围外操作 |
| 退出条件 | 明确：目标状态连续 K 轮稳定；兜底：超过 N 轮未达到稳定则上报 |

### 5. Explore & Map

| 字段 | 默认值 |
|---|---|
| 目标 | 系统性探索未知领域/代码库，建立完整的结构地图 |
| 每轮执行 | 1. 从未探索节点中选择下一个；2. 深入分析该节点；3. 记录发现与关联；4. 更新探索进度 |
| 评估指标 | 已探索节点数 / 总节点估算数；覆盖率百分比 |
| 约束 | 每轮只深入一个节点；不跳跃式探索 |
| 退出条件 | 明确：覆盖率达到目标阈值或无新节点；兜底：N 轮后汇总已知结构并停止 |

## 生成的 prompt.md 格式

```markdown
## GOal

<用户定义的目标，1-3 句话，含成功标准>

## 每轮执行

<每次 loop 应做的具体动作，编号步骤列表>

## 评估（每轮末尾）

<判断进展的方式：打分 / 检查项 / 比对结果>

## 约束

<不能做什么，资源/范围限制；若无则写"无">

## 退出条件

- 明确条件：<用户指定的停止状态>
- 兜底逻辑：<无法判断时 Claude 的行为>

## 过程文档

路径：~/.hskill/init-goal/<goal-slug>/log.md
每轮末尾读取最后一个 Round 条目获取上下文，执行完后追加新条目。
Loop 退出时在同目录生成 summary.md。
```

---

## log.md 每轮条目格式

```markdown
### Round N — YYYY-MM-DD HH:MM

**执行内容：** <本轮做了什么>

**评估结果：** <指标 / 分数 / 检查项结果>

**下一轮建议：** <继续方向，或已达退出条件>
```

---

## summary.md 格式（loop 退出时生成）

```markdown
# GOal Summary — <goal-slug>

## 目标

<复述目标>

## 结果

<最终达成状态，一句话>

## 关键轮次

<哪几轮发生了重要转折，简要说明>

## 退出原因

<明确条件触发 / 兜底逻辑触发 / 用户中断>

## 执行统计

共 N 轮

## 建议（可选）

<若未完全达成，Claude 建议的下一步方向>
```

---

## 输出结果（skill 执行完毕时）

skill 展示：
1. 已生成文件路径
2. 启动提示（用户自选 interval）：
   ```
   /loop <你选择的 interval> $(cat ~/.hskill/init-goal/<goal-slug>/prompt.md)
   ```
3. 提示：如需修改目标重新运行，直接编辑 `prompt.md` 后重启 loop 即可

---

## 不在本 skill 范围内

- `log.md` 和 `summary.md` 的写入——由 loop 运行时的 Claude 负责，prompt.md 中有指令
- interval 选择——用户自决
- loop 运行过程的监控或中断——属于 `/loop` 内置行为
