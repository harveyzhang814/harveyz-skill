# question-me Skill 设计

**日期：** 2026-07-01  
**状态：** 待实现

---

## 背景

任务执行前，用户的指令往往存在隐含假设、边界模糊、或成功标准不清晰。question-me 是一个执行前置 skill，通过结构化问卷帮助用户在开始实现之前澄清指令、对齐预期。

参考 grill-me/grilling 的核心设计哲学：一次一问、每问附推荐答案、按决策依赖顺序推进。

---

## 触发逻辑

### 显式调用
用户输入 `/question-me [任务描述]`，直接进入流程。

### 自动提议
Claude 检测到以下信号时主动提议，等用户确认（y/n）再进入：
- 请求涉及多个可能互相矛盾的目标
- 关键词模糊（"优化一下"、"整理"、"重构"）
- 缺少明确的成功标准或截止范围
- 任务依赖未指明的上下文假设

简单明确的指令（如"运行测试"、"读这个文件"）不触发。

---

## 整体流程

```
触发
 └─ Phase 0：自查（Claude 先读代码/文件，解决能自解的问题）
     └─ Phase 1：意图校准（固定 3 问：目标 / 成功标准 / 范围边界）
         └─ 初始化决策树
             └─ Phase 2：动态深挖（持续追问，直到 Claude 判断 shared understanding 达成，或用户打断）
                 └─ 输出精炼指令摘要（用户确认）
                     └─ 执行
```

---

## Phase 0 — 自查

在问用户之前，Claude 先：
- 读取已提到的文件/目录，推断任务上下文
- 查 git 状态、近期 commit，了解当前进展
- 对能从代码/文档直接确认的问题，自行解答，不占用用户的问答配额

自查结束后简短说明"已了解到 X，剩余不确定的是 Y"，再进入 Phase 1。

---

## Phase 1 — 意图校准（固定 3 问）

每问一次，等用户回答后再问下一问。每问附推荐答案。

| 顺序 | 维度 | 说明 |
|------|------|------|
| 1 | **目标** | 这件事做完，最核心的变化是什么？ |
| 2 | **成功标准** | 怎么判断做对了？（可测量的验收条件） |
| 3 | **范围边界** | 明确不做什么，或哪些东西不能动 |

Phase 1 结束后初始化决策树（见下节）。

---

## 决策树

### 内部格式

每个节点一行，平铺列出：

```
[status]  id=XX  [dep=YY]  节点文本
```

字段说明：
- `status`：`done` / `open` / `infer` / `skip`
- `id`：全树唯一的短 ID（2–3 字母），更新时引用稳定
- `dep=YY`：可选，指向另一节点 id，表示"YY 答完后此节点才可问"；同时用于渲染器推导树结构
- 无 `dep` 的节点为根节点

示例：

```
[goal:done]     id=G              将 tags 拆为 fixed_tags + candidate_tags
[success:done]  id=S              新文章有两字段；旧文章不迁移
[scope:done]    id=SC             只改 Python 脚本；SKILL.md 可补充
[storage:done]  id=ST  dep=SC     ~/.hskill/extract-url/fixed_tags.txt
[format:open]   id=FF  dep=ST     frontmatter 字段结构变化
[compat:infer]  id=CP  dep=FF     旧文章不迁移，新旧格式共存
[review:open]   id=RV  dep=SC     candidate_tags review 流程
```

### 更新规则

- Agent 每次回答后只修改变化的行，不重写全树
- `dep=ID` 在依赖节点变为 done 后，agent **不需要**在行内标注完成状态；渲染器自行查 ID 状态
- `infer` 节点由 Claude 自行填入推断内容，不追问用户

### Phase 2 选题逻辑

> 找所有 `open` 节点中，`dep` 指向的节点状态为 `done` 的 → 按影响面最大的（被最多其他节点依赖的）先问

---

## Phase 2 — 动态深挖

基于 Phase 1 答案和决策树，Claude 逐一追问 `open` 节点，每问附推荐答案。

**停止条件：**
- Claude 判断所有影响执行方向的决策因子已明确（所有 `open` 节点清空或变为 `infer`）
- 用户随时说"够了 / 开始 / 可以了"等打断信号

**不问的节点：** Claude 可以合理默认处理的，直接标为 `infer`，在摘要中透明列出假设。

---

## 输出：精炼指令摘要

Phase 2 结束后，在会话内输出结构化摘要，用户确认后执行：

```
## 任务摘要

**目标：** ...
**成功标准：** ...
**范围：** 包含 ... / 不包含 ...
**关键决策：** （来自 Phase 2 的澄清结果）
**假设：** Claude 自行处理的部分（透明列出）

确认后开始执行。
```

## HTML 渲染

### 两种形式

| 形式 | 维护者 | 用途 |
|------|--------|------|
| 内部树（结构化文本） | Claude | 追问逻辑、优先级判断 |
| HTML 树（可视化） | render_tree.py 自动生成 | 用户查看进度 |

Claude 只维护内部树；HTML 是单向派生，Claude 不手写 HTML。

### render_tree.py

每次 agent 更新内部树后调用：

```bash
python3 render_tree.py "<内部树文本>" /tmp/question-me-tree.html [--open]
```

- `--open`：首次调用时追加，自动在浏览器打开
- 后续调用不追加，页面通过 `<meta http-equiv="refresh" content="3">` 自动刷新
- 纯 HTML + 内联 CSS + 内联 JS，无外部依赖

### 树结构重建算法

1. 无 `dep` 的节点 → 根节点
2. `dep=X` 的节点 → 挂在 X 节点之下
3. 兄弟节点排列顺序 = 内部树文本的出现顺序
4. 跨节点的 `dep` 关系（如兄弟间依赖）→ 渲染为虚线有向箭头

### Card 结构

每个节点渲染为一张 Card：

```
┌─────────────────────────────────────┐
│ ◉  FF                               │
│ ─────────────────────────────────── │
│ Q  frontmatter 字段结构变化          │  ← 节点文本原文（决策主题）
│ A  待回答...                         │  ← 答案（done 后填入用户实际回答）
│ ─────────────────────────────────── │
│ dep: ST                             │
└─────────────────────────────────────┘
```

Q 行显示节点文本原文（描述性，非问句）。完整的问题措辞和推荐答案在底部固定栏展示（仅当前高亮节点）。

A 行的显示逻辑：

| status | A 行内容 |
|--------|---------|
| `done` | 用户的实际答案（绿色文字） |
| `open`（dep 已满足） | `待回答...`（虚线占位，整卡蓝色高亮） |
| `open`（dep 未满足） | `等待 [dep id] 先回答`（灰色半透明） |
| `infer` | Claude 的推断内容（灰色斜体） |
| `skip` | `已跳过`（删除线） |

Card 左边框颜色：`done` 绿 / `open` 黄 / `infer` 灰 / `skip` 浅灰。

### 页面结构

```
┌────────────────────────────────────────────┐
│ question-me  |  任务：<任务标题>            │
│              |  进度：4 / 7  ████░░░ 57%   │
│──────────────────────────────────────────  │
│                                            │
│         [ 可视化树区域，水平居中 ]          │
│         Card 之间实线连父子关系             │
│         dep 跨节点用虚线箭头               │
│                                            │
│──────────────────────────────────────────  │
│ 当前问题：frontmatter 字段结构变化？        │
│ 推荐答案：新增两个独立字段...              │
└────────────────────────────────────────────┘
```

底部固定栏显示当前高亮节点的问题文本和推荐答案，与树图联动。

## 不在范围内

- 跨会话保存问答历史（每次 question-me 会话独立）
- 自动执行任务（摘要确认后由 Claude 自行判断如何执行，不交棒特定 skill）
- 强制用户必须跑完全部 open 节点（用户可随时打断）
- 问答结果写入文件（只在会话内输出摘要）
