**日期**: 2026-06-22

---

## 1. 目的之变：从"诊断这篇文章"到"从这篇文章学到可迁移的思维动作"

旧 skill 的 telos 是**法证式的**：产出一个站得住、防过度归因的"隐含作者"假设，所有闸门都为**压制假阳性**服务。

新 skill 的 telos 是**教学式的（generative 为主）**：从一篇文章里捞出**可迁移、能复用的认知动作**，讲到用户能把它纳入自己的思考与写作工具箱。

这一目的之变，要求改的不只是产出层，还有喂给产出层的分析管线本身。四处法证偏差及其纠正：

| 法证偏差 | 对学习的危害 | 纠正方向 |
|---|---|---|
| ① 错误代价方向：宁丢假阳性 | 假阴性才贵——真正好用的招因"分不清是作者还是体裁"被毙 | 偏向召回可学的招；体裁标配的招照样保留 |
| ② 分析单位是"独特"（甚至 F5 残余过读） | 独特=最不可迁移（怪癖学不走），注意力背对该学的 craft | 单位改为"有效且可迁移的动作"，独特与否不重要 |
| ③ 只反向读（诊断"有什么/缺什么"） | 学不会"怎么做"——缺正向配方 | 新增"生成式重建"：每招 → 意图→步骤→效果的 recipe |
| ④ 判断轴是"归因置信度" | 学习要的是"招何时灵/何时砸" | 判断轴换成有效性/可迁移性/接地程度 |

**该留的（别矫枉过正）**：逐字证据纪律、逻辑图谱、warrant 抽取——全留，它们是防止学习层退化成鸡汤的锚。尤其 **warrant 是已在产却用错地方的金子**：旧版把它当"作者世界观的证据"，新版把它当**发生器**（接受这条默认前提，下面的招就都长出来了）。

**操作信条（执行前内化）：**
- 产出的是**可迁移的认知动作 + 生成它们的根性**，学的单位是剥离了本文内容的招，不是作者怪癖。
- 学习层每条招**必须能回指一张签名卡（证据层）**，不许凭空生成——这是防鸡汤的硬锁。
- generative 为主：重点是"我能怎么用"，批判性判别退为每招一行附注（仍在，非主角）。

## 2. 架构：两层，不是改一层

签名卡**不删，降级为"证据/审计层"**——它的逐字证据、置信度、warrant 是学习层站得住的地基。在它上面**新长一个学习层**。

```
1-evidence.md     残余日志 + 逻辑图谱(+补链日志) + 结构信号     证据底料
2-signature.md    描述层签名卡 = 证据/审计层                    "这文章在干嘛"（防鸡汤的锚）
3-playbook.md     认知动作手册 = 学习层  ★ 头牌产出            "我能怎么用"
4-attribution.md  归因层签名卡（仅模式 B，可选增强）            "这招是不是这作者的"
```

数据流：`证据 → 签名卡(诊断) → 生成式重建 → 手册(教学)`；模式 B 再叠加 `归因(可选)`。

## 3. 管线重瞄（相对旧版的增删）

旧版 7 阶段：1 表层精读 / 2 建图 / 3 结构判读 / 4 两源交叉→描述候选 / 5 归因闸门 / 6 组装置信 / 7 自审。

新版管线：

| 阶段 | 动作 | 相对旧版 |
|---|---|---|
| 1 表层精读 | 残余日志（逐字 span+定位） | **保留**，但读的姿态从"找破绽(tells)"转向**兼读 craft**（作者哪里做得有效、利落） |
| 2 建逻辑图谱 | 节点+边，每条暗连入 WARRANT_LOG | 保留（warrant 在新版是发生器原料，地位升高） |
| 3 结构判读 | 形状/承重/缺边/不对称 | 保留 |
| 4 描述层签名卡 | 两源交叉 → "本文实现了 X" + 逐字实例 | 保留，定位改为**审计层**（防鸡汤锚），不再是终点 |
| **5 生成式重建（新增）★** | 每条签名卡 → 一张**认知动作卡**：意图→步骤→效果的正向配方 + 适用条件 + 回指证据 | **全新**，学习层的核心引擎 |
| **6 发生器蒸馏（新增）★** | 从动作卡集合反推 1–2 条根性（发生器），并标注每招从哪条根性长出 | **全新** |
| 7 归因闸门 | Patternicity / Agenticity / 对立解释 / 对抗 pass / 留出验证 | **降级为模式 B 可选增强**，不再是管线脊梁；机器本身完整保留（旧版 §阶段5 的子任务隔离设计原样搬过来） |
| 8 交付前自审 | 自审清单（含新版条目） | 保留+改写 |

**基线语义翻转**：法证里基线是**减法滤镜**（扣掉体裁信号以隔离作者）；教学里基线是**加法样本**（多看几个人怎么使同一招 → 把招泛化得更牢）。同一输入，作用相反。模式 B 仍可用基线做减法（归因层需要），但**学习层用基线做加法**——若用户提供了基线，手册里对应招可附"别处也这么使"的泛化注脚。

**错误代价翻转**：阶段 4→5 不再因"可能是体裁惯例"丢弃候选。一个常规但有效的招照样进手册。法证的丢弃逻辑只在模式 B 的归因层（阶段 7）生效。

## 4. 学习层结构：发生器优先（已与用户确认）

`3-playbook.md` 不是平铺 N 张招卡，而是发生器优先、招卡为演示：

```
# {标题} — 认知动作手册
**来源**: <article_path>  **模式**: A/B  **日期**: YYYY-MM-DD

## 一、这套思维的发生器（先学这个）
   根性镜头：1–2 条 —— "作者把 X 看成 Y / 默认 Z 不证自明"（来自 WARRANT_LOG 蒸馏）
   为什么先学它：学会镜头能自己再生出下面的招；只背招，换题就忘。

## 二、认知动作 ×N（每条 = 发生器落到一个具体操作）
   [认知动作卡 schema 见 §5]

## 三、迁移练习汇总
   把各招的练习 prompt 汇成一张清单，供用户拿自己的题目逐条练。
```

## 5. 认知动作卡 schema（学习层输出契约）

```
认知动作 #k：[可迁移的招名 —— 剥离本文内容，写成通用操作]

  这招替你干什么活:  [它在论证/表达里完成的认知工作；为何有效]
  怎么自己跑:        [① … ② … ③ … 可复现的正向步骤]
  本文怎么使的:      [≥1 处逐字实例 + 定位（来自签名卡）]
  该学还是该防:      [一行附注：✓纳入工具箱 / ✗识破防御 / ~双刃 + 本文判定]
  从哪条根性长出:    [回指 §一 发生器的哪条镜头]
  回指证据:          [签名卡 #k —— 强制，不许无锚生成]
  迁移练习:          [一个让用户拿自己的题目套用此招的 prompt]
```

**鸡汤锁**：`回指证据` 为空的动作卡不许写出——每招必须钉到一张签名卡的逐字实例。

**判断轴**：动作卡不评"归因置信度"，评**有效性 / 可迁移性 / 接地程度**。`该学还是该防` 那行承载"何时灵/何时砸"。

## 6. 流程控制（`--pass`）与模式

| 参数 | 行为 | 依赖 |
|---|---|---|
| `--pass 1` | 阶段 1–3 → `1-evidence.md` | 无 |
| `--pass 2` | 阶段 4 → `2-signature.md` | `1-evidence.md` 已存在 |
| `--pass 3` | 阶段 5–6 → `3-playbook.md` | `2-signature.md` 已存在 |
| `--pass 4` | 阶段 7（归因闸门）→ `4-attribution.md` | `2-signature.md` 已存在 + 模式 B + 重新提供原文与基线路径 |
| 无参数 | 跑到 `3-playbook.md`（模式 A/B 都到手册）；模式 B 再加 `4-attribution.md` | — |

- **头牌产出 `3-playbook.md` 建在描述层（模式 A 即可）之上**——学一个招"怎么用"不需要先归因到作者。这让头牌对所有用户可用。
- 模式 A：evidence → signature → **playbook**（默认终点）。
- 模式 B：额外产出 `4-attribution.md`，并在 playbook 里用基线做加法泛化。
- "无基线禁止归因"红线**仅约束 `4-attribution.md`**，不约束学习层。

文件名变更：旧 `2-descriptive.md` → `2-signature.md`；旧 `3-attribution.md` → `4-attribution.md`。

## 7. 输入 / 硬停机 / 降级（沿用旧版，微调）

- 输入：本地 `.md`/`.txt` 全文；模式 B 另需 2–3 篇同类型基线。
- 硬停机：少于约 300 字或片段 / 纯数据无论述 / 非本地文本 → 拒绝并说明。
- 降级：想归因但无基线 → 学习层照常产出（模式 A），仅 `4-attribution.md` 不产，明确告知"无基线不归因，但手册照常给"。
- 配置：`~/.hskill/extract-cognition/config.json`，字段 `output_dir`，默认 `~/Documents/cognition`；写入用 `$HOME` 展开。
- **空结果是合法结果**：无可锚定的招 → 如实交付，不为产出降标准。

## 8. 上限声明（学习层版本，每次必附）

> 本手册抽取的是**本文实现的、可迁移的认知动作**及其背后的默认前提，作为你可学习/可警惕的对象。每招都锚定了本文逐字实例；但"这是不是该作者的稳定特质"属于归因问题，仅在模式 B 的 `4-attribution.md` 中、且有体裁基线时才作有限主张。手册不等于为这些招背书——`该学还是该防` 一栏给出本文语境下的判定。

## 9. 失败模式（在旧版 F1–F5 上增补）

沿用 F1 模式强加 / F2 流畅过度归因 / F3 体裁冒充 / F4 置信膨胀 / F5 残余过读，增补两条学习层专属：

- **F6 — 鸡汤化**：动作卡写得漂亮却钉不到签名卡逐字实例。*纠正*：删；强制回指证据。
- **F7 — 不可迁移**：招卡仍绑死在本文内容（出现了文章的专有名词/具体话题才成立）。*纠正*：重写成内容无关的通用操作，迁移练习能换题套用才算过。

## 12. 暂不做（YAGNI）

- 交互式陪练 / 打分循环（手册只预置静态练习 prompt）。
- URL 抓取、PDF 解析。
- 跨语料 / 跨文章作者画像。
- 与元框架/语言框架提取 skill 的衔接。
- 自动寻找体裁基线。
