# Executing Plans - 计划执行

## 概述

加载计划，批判性审查，执行所有任务，完成后报告。

**开始时宣布：** "正在使用 executing-plans 技能执行此计划。"

## 前置要求

**必须先执行：** `using-git-worktrees` 技能创建隔离工作区

## 执行流程

### Step 1: 加载和审查计划

1. 读取计划文件
2. 批判性审查 — 识别任何问题或顾虑
3. 如有问题：开始前向用户提出
4. 如无问题：创建 TodoWrite 并继续

### Step 2: 执行任务

对于每个任务：
1. 标记为进行中
2. 精确遵循每个步骤
3. 运行指定的验证
4. 标记为完成

### Step 3: 完成开发

所有任务完成后：
1. 宣布使用 `finishing-a-development-branch` 技能完成此工作
2. 验证测试
3. 呈现选项
4. 执行选择

## 两阶段 Review

### 阶段 1: 规格合规性

检查计划中的每个要求是否有对应实现：
- 浏览规格的每个部分
- 能指向实现它的任务吗？
- 列出任何空白

### 阶段 2: 代码质量

检查代码是否：
- 遵循项目现有模式
- 有适当的错误处理
- 有必要的测试覆盖
- 提交信息清晰

## 何时停止并寻求帮助

**立即停止当：**
- 遇到阻碍（缺少依赖、测试失败、指令不清）
- 计划有关键空白无法开始
- 不理解某条指令
- 验证反复失败

**宁可问清楚不要猜。**

## 何时返回早期步骤

**返回审查（Step 1）当：**
- 用户根据反馈更新了计划
- 基本方法需要重新思考

**不要强行通过阻碍** — 停止并问。

## 关键约束

- 审查计划要先批判性思考
- 精确遵循计划步骤
- 不要跳过验证
- 在隔离工作区执行
- 从不在 main/master 分支直接实现（除非明确用户许可）

## 与其他技能配合

**必需工作流技能：**
- **using-git-worktrees** — 设置隔离工作区
- **writing-plans** — 创建此技能执行的计划
- **systematic-debugging** — 遇到 bug 时
- **finishing-a-development-branch** — 完成后清理

## 常见错误

| 错误 | 正确做法 |
|------|----------|
| 未隔离就执行 | 先用 using-git-worktrees |
| 不审查直接执行 | 先批判性审查计划 |
| 跳过验证步骤 | 每个验证都要跑 |
| 遇到阻碍继续猜 | 停止并问清楚 |
| 主分支直接改 | 必须隔离分支 |
