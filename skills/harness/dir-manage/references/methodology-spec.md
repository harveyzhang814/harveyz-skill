# DIR_METHOD.md 格式规范

`DIR_METHOD.md` 是自定义方法论的定义文件，仅在目录使用非内置方法论时才需要。内置方法论（Diátaxis、角色分类法）通过 `INDEX.md` 的 YAML frontmatter 声明，无需此文件。

---

## 完整格式

````markdown
# 方法论名称

一句话说明这套方法论的核心原则（供人类理解，不影响 skill 解析）。

## 配置

```yaml
index_file: INDEX.md          # 索引文件名，默认 INDEX.md
managed_dir: docs/            # 被管理的目录路径（相对于仓库根）
```

## 分类（categories）

列出所有分类，每个分类包含：
- **目录名**：实际的子目录
- **定义**：一句话，说明读者处于什么状态才会来这里
- **包含（includes）**：只放这类内容
- **排除（excludes）**：不放这类内容

### [分类A名称] (`目录名/`)

定义：……

包含：……

排除：……

### [分类B名称] (`目录名/`)

……（重复上面的结构）

## 分类决策流程（classification）

用问答树描述如何判断一个文档属于哪个分类。

```
问题1？
  是 → 分类A
  否 → 问题2？
          是 → 分类B
          否 → 分类C
```

## 扩展分类（extensions，可选）

说明什么情况下可以在四个基础分类之外新增目录，以及新增目录需满足的条件。

````

---

## 最小可用格式

如果方法论很简单，`categories` 和 `classification` 是必需的，其余字段可省略：

````markdown
# 方法论名称

## 分类

### 分类A (`目录A/`)
定义：……  
包含：……  
排除：……

### 分类B (`目录B/`)
……

## 分类决策流程

```
是X类内容？
  是 → 目录A/
  否 → 目录B/
```
````

---

## 注意事项

- `DIR_METHOD.md` 描述的是**这个目录自己的结构规则**，不是通用文档
